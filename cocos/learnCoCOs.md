## 一、游戏开发者成长四阶段

| 阶段 | 核心目标 | 典型行为 | 产出 |
|------|---------|---------|------|
| **1. 新手** | 跑通流程 | 跟着教程做，让东西动起来 | 第一个场景、第一个移动的方块 |
| **2. 熟练开发者** | 独立完成功能 | 不依赖教程，了解引擎常用模块 | 一个完整可玩的小游戏Demo |

### 阶段2：Cocos Creator 编辑器与核心机制

#### 编辑器基础操作
- 资源管理器：导入资源、管理场景和脚本
- 场景编辑器：搭建场景、摆放节点
- 层级管理器：管理节点父子关系、渲染顺序
- 属性检查器：修改属性、绑定脚本

#### 节点与组件系统
- 节点(Node)：场景中每个物体都是一个节点
- 组件(Component)：赋予节点功能（Sprite显示图片、Button可点击、脚本自定义逻辑）
- 预制体(Prefab)：将节点保存为模板，动态生成实例

#### 脚本生命周期

| 生命周期函数 | 触发时机 | 典型用途 |
|------------|---------|---------|
| `onLoad()` | 组件加载时 | 初始化数据、获取组件引用 |
| `start()` | 第一次激活前 | 依赖其他节点的初始化 |
| `update(dt)` | 每帧调用 | 移动、逻辑判断、实时响应 |
| `lateUpdate()` | update之后调用 | 相机跟随 |
| `onDestroy()` | 节点销毁时 | 清理监听器、释放资源 |

#### 事件系统（EventTarget）
Cocos 的原生解耦机制——节点和组件通过事件通信，避免硬引用。

| 方法 | 说明 | 典型用途 |
|-----|------|---------|
| `node.on(eventType, callback)` | 监听事件 | 玩家死亡时 UI 更新分数 |
| `node.emit(eventType, args)` | 触发事件 | 碰撞检测后通知分数变化 |
| `node.off(eventType, callback)` | 移除监听 | 节点销毁前清理，防止内存泄漏 |

#### 场景管理与生命周期
- **场景切换**：`director.loadScene('场景名')` 加载并切换场景
- **预加载**：`director.preloadScene('场景名')` 后台预加载，减少切换卡顿
- **场景数据传递**：通过常驻节点或 `director.loadScene('场景名', callback)` 传参
- **常驻节点**：`game.addPersistRootNode(node)` 使节点在切换场景时不被销毁，用于全局管理器（GameManager、AudioManager 等）

#### 音频系统
- **AudioSource 组件**：挂载到节点上播放音频
- **AudioClip 资源**：音频文件资源（支持 mp3、ogg、wav）
- **背景音乐**：循环播放，常驻节点管理，切换场景不中断
- **音效**：短音频，可叠加播放，通常绑定到具体事件（按钮点击、碰撞等）

**验收标准：**
- [ ] 能独立创建场景，添加节点并摆放位置
- [ ] 能创建脚本组件并实现简单的移动逻辑

### 阶段4：动画系统

| 知识点 | 说明 | 使用场景 |
|-------|------|---------|
| Animation 组件 | 挂载到节点上管理动画 | 所有动画的基础 |

| 动画事件 | 在动画特定帧触发函数 | 攻击判定帧、脚步声 |
| 缓动系统(tween) | 代码驱动的补间动画 | 弹窗效果、数字变化动画 |
| 序列帧动画 | 逐帧切换图片 | 爆炸特效、火焰动画 |

### 阶段6：微信小游戏对接

#### 发布流程
1. 注册微信小游戏账号，获取 AppID
2. 下载微信开发者工具
3. Cocos 构建发布 → 平台选「微信小游戏」
4. 真机预览：扫码测试

#### 包体管理（核心难点）
- **4MB限制**：首包不能超过4MB
- **远程资源加载**：图片/音效上传到服务器，动态下载
- **MD5 Cache**：解决缓存更新问题
- **资源缓存管理**：`wx.downloader.cleanAllAssets()`

#### 微信 SDK 调用
- `wx.login()`：获取用户登录凭证
- `wx.getUserInfo()`：获取用户信息
- `wx.shareAppMessage()`：分享游戏
- `wx.createBannerAd()`：Banner广告
- `wx.createRewardedVideoAd()`：激励视频广告

**验收标准：**
- [ ] 能成功发布并在手机上预览
- [ ] 能实现远程资源加载
- [ ] 能调通至少一个微信 API

## 三、深入技术学习计划（资深开发者必经之路）

### 资深 vs. 熟练 能力对比

| 能力维度 | 熟练开发者 | 资深开发者 |
|---------|-----------|-----------|
| **性能优化** | 知道DrawCall越低越好 | 会用工具分析瓶颈，应用动态图集、对象池 |
| **架构设计** | 逻辑写在update里 | 设计状态机、事件系统，模块解耦 |
| **渲染与Shader** | 只用内置Material | 编写自定义Shader，理解渲染管线 |
| **动画系统** | 会用Animation组件 | 集成Spine骨骼动画，掌握程序化动画 |
| **微信平台** | 能发布、调基础API | 解决包体管理，接入开放数据域，优化缓存 |


### 第一层：核心深化（必须掌握）

#### 1. 性能优化（重中之重）
- **渲染优化**：理解合批原理，学会使用动态图集和静态图集减少DrawCall
- **内存管理**：使用对象池避免GC卡顿
- **资源加载**：掌握远程资源加载、分包加载、AssetManager

#### 2. 渲染管线与自定义效果
- **内置渲染管线**：理解前向渲染流程
- **自定义渲染管线**（Cocos 3.6+）：添加后处理特效
- **Shader编写**：实现动态流光、边缘光、溶解等特效

#### 3. 高级动画技术
- **程序化动画**（Cocos 3.8+）：代码实时计算角色姿态
- **骨骼动画集成**：Spine/DragonBones，实现换装、挂点
- **复杂Tween应用**：并行/序列控制，制作复杂UI过场


### 第二层：高阶进阶（按需选择）

#### 1. 微信平台能力深度集成
- 性能深度分析工具
- 1GB缓存空间管理
- 好友排行榜（开放数据域）、实时语音、试玩广告

#### 2. 数据驱动与架构设计
- **状态机（FSM）**：管理角色和游戏状态
- **事件系统**：实现模块解耦
- **数据驱动UI**：UI根据数据模型自动刷新

#### 3. 3D游戏开发入门
- 3D核心概念：场景、摄像机、光照、材质、网格
- 3D物理系统
- 3D项目优化：模型面数、LOD、光照烘焙


### 第三层：专家领域（长期探索）

- **底层渲染与图形API**：WebGL 2.0 / WebGPU，Metal / Emscripten GLX模式
- **WebAssembly (Wasm)**：将C/C++库编译成Wasm，获得原生性能
- **自定义引擎底层**：修改Cocos引擎C++核心层并重新编译
