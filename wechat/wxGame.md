# 微信小游戏开发入门指南

## 1. 游戏引擎选择

| 引擎 | 特点 | 适用场景 |
|------|------|----------|
| **Cocos Creator** | 包体小（首包1-2MB）、2D性能出色、中文友好、市场占有率高 | 2D或轻度3D游戏（**最稳妥推荐**） |
| **LayaAir** | 3D渲染性能强悍、支持一键发布多平台 | 中重度3D游戏 |
| **Unity/团结引擎** | 有UE经验的开发者最熟悉、移植现有项目方便 | 已有Unity项目需移植（注意包体优化） |
| **微信原生开发** | 控制度最高、包体最小、但工作量大 | 核心玩法极简单的项目 |

## 2. 美术动画资源获取

### 免费素材平台

- **Kenney.nl**：海量CC0协议素材（2D/3D/UI），可免费商用
- **OpenGameArt.org**：老牌社区，资源丰富（注意授权方式）
- **Mixamo**：Adobe免费3D角色动画库，可导出FBX用于Unity

### AI生成工具

- **腾讯混元游戏视觉生成平台**：工业级，文字/草稿生成高质量素材
- **Meowart / PixelLab**：像素风格专用工具，支持完美像素和角色一致性

## 3. 对接适配微信小游戏流程

### 步骤一：环境与账号

1. 注册**微信公众平台**小游戏账号（个人主体功能受限，内购需企业主体）
2. 下载安装**微信开发者工具**

### 步骤二：引擎开发与导出

1. 在所选引擎中完成游戏开发
2. 使用引擎的**一键发布**或专属SDK导出代码包

### 步骤三：真机预览与提审

1. 在微信开发者工具中创建小游戏项目
2. 生成预览二维码，真机测试调试
3. 后台提交代码审核

### 核心开发要点

- **包体限制**：首包不得超过**4MB**，非核心资源需采用**远程加载**或分包加载
- **API使用**：分享、支付等功能必须调用微信官方 `wx.` 系列API
- **商业化**：
  - IAA（广告变现）：接入微信广告插件，使用激励视频广告
  - IAP（内购）：接入微信支付实现道具购买

## 快速总结

| 事项 | 推荐方案 |
|------|----------|
| 新手首选引擎 | Cocos Creator |
| 免费2D/UI素材 | Kenney.nl |
| 3D动画 | Mixamo |
| 像素风AI生成 | Meowart / PixelLab |
| 首包大小 | 控制在4MB以内 |
| 商业化 | 广告（IAA）或内购（IAP） |
